# Agent 研判出题生产规则

> 变更记录：暂无
> 生产培训录屏：https://meetchances.feishu.cn/minutes/obcneo62om2272w1xzml2j61

## 需求概述

由专业领域的复杂场景给大模型出题，用于后续考察模型的能力表现，能否作为领域专家解答**真实客户/用户场景**下的复杂问题。

**一句话翻译**：作为 AI 在该领域的老师给大模型出自己工作中实际会遇到的题，需确保这道题目自己也会做，和我们提供的工具共同高效完成一道题的生产，各位专家需要对工具辅助给出的信息做确认和优化。

**规则变更概述**：

1. 出题范围扩大：新增 P0 领域
2. 附件格式要求细化（拒收音频/视频/截图/小语种附件/AI合成附件或AI处理的附件）
3. 做题流程变化（参与程度提升、新增关键步骤和 check list、预审辅助）

---

## 我们想要什么样的数据？

**理想态与原则**：

1. ✅ 附件包含丰富/复杂信息，题目为全面分析总结、出方案等
2. ✅ 附件信息相对简单 or 提供数据，题目为多约束多条件复杂计算分析推理
3. ✅ 附件和题目都复杂 -- ok
4. ❌ 附件信息价值低且题目潦草或存在显著真实性问题：不 OK

---

## 关键定义

1. **短 Query**：即题目/任务本身，给大模型提问题时的文字版描述，真实、清晰
2. **长 Query**：即题目/任务本身的信息丰富版，真实、清晰、详细
3. **附件**：即做题时参考的、需要单独上传的文件，目的是提供全面、权威的信息参考或任务执行过程指导，对能做出这道题起到直接增益
4. **Metadata**：即一道题目的附属信息，非题目正文内容，而是题目的"档案信息"，例如（做题关键步骤、做题时长、附件内容概括等），大部分由工具辅助生成，但可能存在提取失败/错误的情况

## 详细规则

### 1. UUID 上传环节 & 预审1

UUID 携带着领题信息（P0、P1领域，及附件格式要求）

⚠️ 注意避免以下错误，否则会被预审1拦截：

1. UUID 填写内容错误（例如多一位数字、少一位数字等）
2. UUID 被重复使用

### 2. 附件 & 短 Query 上传阶段

| 具体操作 | 操作要求 | 核心标准 | 补充标准 |
|----------|----------|----------|----------|
| 1. 选择领域 | 单选-领域一二级类目 | -- | 当出现跨领域类目的题目时，根据最核心的任务目标去进行选择 |
| 2. 选择附件来源 | 单选-公开/个人渠道 | 个人渠道：非上传互联网的、非AI生成的、工作中真实的<br/>公开渠道：上传至互联网其他人可获取到的 | 当附件均为个人渠道时选择个人渠道，其他情况均选择公开渠道 |
| 3. 上传附件 | 上传2-8个附件 | ✅ 接收：高价值、或高信息密度附件<br/>🤔 辩证接收：英文附件若质量达标可接收<br/>❌ 拒收：音频/视频/截图/小语种<br/>❌ 拒收低价值：信息密度低、AI合成文档 | 附件数量区间：2-8<br/>附件大小建议在：10M-500M<br/>单次上传的附件不能大于300页<br/>拒绝上传整本书类型的附件 |
| 4. 撰写短 Query | 人工撰写，严禁AI生成 | 领域匹配、出题场景真实、任务目标清晰、有难度、产物要求明确，字数预计300字以内 | 注意时效性和时间戳要求 |
| 5. 撰写附件关联性说明 | 人工撰写，严禁AI生成 | 覆盖到所有附件，说清附件之间的佐证关系以及为解决什么问题而服务 | 英文附件需要补充说明必须参考英文附件的原因 |

### 3. Query 相似度校验(1.5) & 预审2

完成相关信息填写后可触发预审：

1. 预审1.5不通过时，会给出拦截信息，需修改 query
2. 预审1.5通过后会自动进入预审2，预审2的结论提供审核建议，用于自行比对识别问题
3. 触发次数上限：3次

| 考察项 | 标准 | 检查逻辑 |
|-------|------|----------|
| 领域相关性 | 附件+Q完全属于所选一二级领域 | ⚠️ 是否基于最核心的任务目标去进行选择了？ |
| 场景真实性 | 现实合理+专业可行+实际应用 | ⚠️ 这个场景在实际工作中可能遇到吗？ |
| 内容质量 | 附件信息价值高、可信度强、Query专业性强/有难度 | ⚠️ 是否存在低信息价值附件<br/>⚠️ 附件之间、以及附件和Q之间核心事实or数据口径是否对齐？ |
| Q和附件关联性 | 需要全部附件 & 仅需这些附件 | ⚠️ 移除任意附件，Query还能完整解答吗？<br/>⚠️ 仅凭这些附件，Query能完全解答吗？ |
| 其他检验项 | 合规性、AI附件校验、语言校验 | -- |

### 4. 长 Query & 关键步骤阶段

#### Query 选择/改写/构造

生产工具1会基于前置信息生成3条长Query，按照以下要求完成Query选择：
若不选-需勾选理由，若选择-需判断是否进行改写，若都不选-需进行改写。

判断路径：选不选？为啥不选？改不改？改成什么样？是否新构造一个？

⚠️ 需注意：
- 若3条都判断可用，也需要优中选优最终 pick 一个，再判断是否需要改写
- 若3条都不可用，需分别备注不可选择的原因，并构造符合要求的 query

**不选择原因**：
1. **难度过低**：人类领域专家解决该题目的预估时长低于8h视为难度过低
2. **附件与Q关联性差**：只需利用其中部分附件，或已提供附件的内容不足以解决该题目
3. **不符合真实场景**：由于视角错误、重点偏离等，导致题目不符合领域下实际会遇到的决策/研究场景
4. **非最优Q**：符合选择要求，但相对弱于最终可提交的最优 query
5. **其他**：请备注概括原因

**改写/创作要求**：内容符合提交要求但表述、结构、细节内容要求时需改写/创作：

1. **文风/逻辑**：从实际工作场景出发描述问题、给出明确任务即可
2. **约束/目标**：Query中目标和约束条件尽可能明确
3. **必需信息点**：注意判断是否全面，若缺失需手动补充

#### 关键步骤填写 & 预审3

触发次数上限：1次

**填写要求**：
1. **逻辑性与顺序**：需符合做题逻辑，步骤间有逻辑递进关系
2. **关键步骤任务量**：每个步骤是一个独立的分析或判断任务
3. **专业性**：是否体现领域核心思维和专业方法
4. **表述方式**：同领域内不同题目的关键步骤需要 by 题目撰写

### 5. Metadata 调优阶段

生产工具2会生成 metadata，需要人工进行确认、调整、补充。

**生成信息范围**：三级类目、任务概括、附件格式标签、附件内容概括、产物格式标签、产物内容概括、做题关键步骤、打分 checklist

**手填信息范围（必填项）**：专家工作年限、预估做题时长、生产数据耗时

**备注（选填项）**：可贴上附件可核验的来源链接，或其他补充说明

### 6. 打分 Checklist

工具会预生成，罗列这道题**必须满足的所有强制要求**。即产物必须达成的、非黑即白的客观标准，少做到一条就算做错。

每一条 check list 的项应遵循原子性，不可拆分性，不要在一个项里check多个子任务/要求。

**判断逻辑**：
1. 完整性「有没有缺少的强制要求」：若有，则补充
2. 性质「每一条是否均属于强制要求」：如果产物没做到某一条，能直接判定产物是错的/不合格的吗？

**示例**：
- 题目：写一封50字以内的道歉信向朋友道歉迟到，必须说清迟到了多久，不能用"对不起"三个字
- Checklist：
  - ✅ 字数50字以内（保留）
  - ✅ 提到了具体迟到了多久（保留）
  - ❌ 语气真诚，态度诚恳（删除，"真诚"是主观判断）
  - ➕ 不能用"对不起"三个字（补充，被遗漏的硬约束）

## 规则尺度细节

### 附件质量要求
网页资讯类信息作为附件时，对质量、信息密度、专业性、权威性等有更高要求。需达到行业或某单项维度深度解析文章的层次。

不接受普通咨询类型的信息作为附件上传（例如搜狐网页、东方财富网等渠道），该类信息会被视为低质量附件拒绝。

### 附件时效性要求
当 query 任务为投资决策等强时效性任务时，需注意附件的时效性是否符合题目需求。

### 附件中隐私信息的界定
以「信息是否具有个人风险性」为核心标准，而非「是否涉及个人」，区分职业公开信息和个人敏感信息。

| 维度 | 可接受（通过） | 会拒绝 |
|------|------------|--------|
| 判断标准 | 信息本身具有公开性或推广性质，来源是公开渠道或当事人主动公开 | 信息一旦泄露可能导致当事人遭受身份盗用、歧视、骚扰等实质伤害 |
| 典型场景 | 职业联系方式、公开发布的署名信息 | 证件内容、医疗健康信息、个人隐私账号 |
| 示例 | 附件为某券商研报，封面页有"分析师：张三，电话：010-88888888" | 身份证正反面扫描件、医院诊断证明含患者隐私信息、背景调查报告含银行流水 |
