一、色彩系统：从“显示颜色”升级为“传递情绪”
1. 主色调克制
   整体 UI 用大面积的白色、浅灰、暖灰作为背景，只让油漆色卡本身成为页面上最鲜艳的元素。

导航栏、按钮、文字不用高饱和蓝红绿，改为炭黑、深灰、低明度品牌色。

2. 色彩过渡与氛围
   每个色卡的背景不用纯白色，而是用极浅的同色系渐变，例如展示米黄色油漆时，卡片背景是 2% 的米黄渐变到白，形成“浸润感”。

页面整体的底色也可以根据当前浏览的色系发生微妙变化（暖色系页面微微偏暖，冷色系微微偏冷）。

二、字体与排版：克制、留白、节奏
1. 字体选择
   标题用无衬线现代字体，如系统原生“苹方”或“Inter”，字重控制在 Medium/Semibold，不用粗黑。

正文字号不要大，14–15px 即可，行高 1.6–1.8，字间距略宽松（0.3–0.5px）。

2. 留白哲学
   卡片的 padding 和 margin 至少是常规的 1.5 倍，例如 24px 起，让元素之间“透气”。

标题和正文之间用空白分隔，而不是线条。

3. 信息层级
   每个色卡展示：色块（70%面积）+ 颜色名称 + 品牌/规格，价格放在详情页或悬浮时才出现，不在列表页铺满数字。

三、动效与交互：微交互传达精致
1. 悬浮态
   鼠标悬停在色卡上时，色块轻微放大（scale 1.03–1.05），并升起柔和阴影（多层 box-shadow 叠加）。

卡片本身有 200–300ms 的过渡动画，缓动函数用 cubic-bezier(0.4, 0, 0.2, 1)。

2. 滚动视差与渐现
   页面滚动时，色卡不是直接弹出，而是从下方淡入向上平移（Intersection Observer + translateY + opacity）。

首屏背景可以有一个极慢移动的抽象色块（CSS 动画），模拟油漆在液体中流动的感觉，但不抢眼。

3. 导航与页面过渡
   切换页面时（如从列表到详情），可以用 Astro 的 View Transitions API 实现共享元素渐变过渡，比如色块从卡片位置平滑移动到详情页的大图位置。

四、质感与细节：模拟实体油漆的触感
1. 色块质感
   色块不能是一个扁平的 <div>。必须叠加微妙的噪点纹理（用 SVG 或 Canvas 生成极小颗粒），模拟油漆墙面的漫反射感。

甚至可以叠加一层极浅的线性渐变（从亮到暗 2%），模拟光照在墙面的微妙变化。

2. 图片处理
   实景案例图片统一加** 0.5px 的白色内边框和微阴影**，模拟画册印刷品的边框。

所有图片加载前用骨架屏或模糊占位图，避免突然弹出。

3. 图标与装饰
   所有小图标（箭头、电话、微信）用极细线条风格（Stroke 1.5px），不用实心填充。

列表之间不适用分割线，用间距 + 浅色背景交替区分。

五、内容呈现：少即是多，但所见即所得
1. 颜色展示方式升级
   除了色卡，增加“大面积展示模式”：点击某个颜色后，整个视口背景都变成该颜色（或该颜色的渐变），中间只留一个白色的 X 关闭按钮和极简的文字信息，让用户沉浸式感受颜色占满墙面的效果。

这就是“高级感”的核心：让用户感受颜色，而不是浏览颜色。

2. 案例的叙事感
   实景案例不用瀑布流，而是用大尺寸横向滑动卡片，每张卡片占屏幕宽度 80%，有标题、简短描述、所用油漆标签，像翻阅高端家居杂志。

3. 联系方式的艺术化处理
   底部联系栏不用传统的蓝色链接，而是设计成一个悬浮胶囊：磨砂玻璃效果（backdrop-filter），内嵌微信图标和手机图标，文字极简，悬浮在右下角。

六、高级感的具体落地建议（直接加进你的 Skill）
全局 CSS 变量：定义一套低饱和、高明度的颜色系统，UI 色和产品色严格分离。

字体引入：通过 Google Fonts 或 CDN 引入 Inter 或保留系统苹方，字重只用 300/400/500。

纹理图层：在 public/ 下放一个 200x200 的半透明噪点 PNG，用 CSS 作为色块的 background-image 叠加。

滚动动画：用 @astrojs/aos 或自己写一个简单的 Intersection Observer 指令。

View Transitions：Astro 已原生支持，在 BaseLayout 中添加 <ViewTransitions /> 组件，并给每个色卡加 transition:name 属性，实现平滑页面切换。

磨砂玻璃效果：联系栏和导航栏用 backdrop-filter: blur(12px) + background: rgba(255,255,255,0.7)。
